## 12.5. System Guidelines – Task Management System

### Kiến trúc tổng thể (Clean Architecture – Domain First)

- **Domain:** Chứa logic nghiệp vụ, Entities, Value Objects, Domain Services (không chứa thư viện ngoài). Mọi rule nghiệp vụ kiểm tra bên trong phương thức entity.
- **Application:** CQRS bằng MediatR. Validations với FluentValidation. DTOs mapping.
- **Infrastructure:** EF Core, Redis, RabbitMQ, Hangfire. Repository triển khai interface của Application.
- **API:** Minimal API, bảo mật JWT (Access Token 15 phút, Refresh Token xoay vòng 7 ngày), Policy-based Authorization, Global Exception Handler.

### Frontend Guidelines (React + TypeScript, Ant Design)

- **Stack:** React 18+, Ant Design 5+, Redux Toolkit, React Query, React Router v6, Axios, SignalR.
- **Quy tắc:** Form validation phía frontend, API call luôn qua custom hook. Không chứa business logic ở component.
- **Thiết kế UI/UX:** Dark theme (primary #1890ff, danger #ff4444). Responsive ưu tiên Desktop, co giãn tốt trên Tablet. Mọi thay đổi task được audit log và không thể xóa.

### CI/CD & Deploy

- Nhánh `main`, `develop`, `feature/*`. Áp dụng Pull Request rules.
- **Deploy:** Backend chạy Docker container trên Ubuntu Server. DB PostgreSQL 17. Redis/RabbitMQ qua container. Frontend serve qua Nginx. Rate limit 100 req/phút/IP. Chống brute-force khóa tài khoản 10 phút.
